# 🧠 Real-time Engine

Nalix keeps real-time state consistent by pairing a deterministic listener with lightweight client helpers.

- **Connection orchestration** – `TcpListenerBase` feeds every accepted socket into `ConnectionHub`, which shards connections, tracks usernames, evicts anonymous connections when limits hit, and exposes `Statistics` for telemetry.
- **Packet dispatch** – `PacketDispatchChannel` reads metadata generated by `PacketMetadataBuilder`, publishes inbound frames through `PacketContext<IPacket>`, and pushes them through `MiddlewarePipeline<TPacket>` before calling handlers such as `PingHandlers`.
- **Guarantees** – `RequestExtensions.RequestAsync` registers a predicate before the request leaves, avoiding race conditions even when a server replies fast. `ControlExtensions.PingAsync` measures RTT and optionally syncs time via `Clock.SynchronizeTime`, so you can align clocks between devices.
- **Client feedback** – `IoTTcpSession` raises `OnMessageReceived` with pooled buffers, fires `OnDisconnected`/`OnReconnected`, and uses `TransportOptions` (heartbeat, retries) to avoid stalls.

!!! note
    Handlers (marked with `[PacketController]` and `[PacketOpcode]`) receive `PacketContext` instances that expose the raw `IPacket`, the owning `IConnection`, and a `PacketSender`. Use `PacketSender.SendAsync` to reply with the correct cipher/compression flags per handler attributes.
